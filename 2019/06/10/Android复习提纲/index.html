src 目录：源代码目录，允许所有用户修改的 java 文件和用户自己添加的 java 文件都保存在这个目录中

gen 目录：用来保存 ADT 自动生成的 java 文件，例如 R.java 或 ADL 文件

assets 目录：用来存放原始格式的文件，其中的文件不能被 R.java 文件索引，只能以字节流读取

bin 目录：保存了编译过程中产生的文件，以及最终产生的 apk 文件

res 目录：资源目录，Android 程序所有的图像、颜色、风格、主题、界面布局和字符串等资源都保存在旗下的几个子目录中

android.jar 文件：Android 程序所能引用的函数库文件，Android 系统所支持 API 都包含在这个文件中

android-support-v4.jar 文件：是 Android 兼容性包（Compatibility package），让一个在 Android 4.0 上开发的应用可以在 API level 4 的安卓系统中工作

proguard.cfg 文件：提供 ProGuard 工具进行代码优化和代码混淆使用的配置文件

project.properties 文件：记录了 Android 工程相关设置

AndroidManifest.xml 文件：是 XML 格式的 Android 程序声明文件，包含了 Android 系统运行 Android 程序前所必须掌握的重要信息

  1. 安卓的配置文件作用，基本组成标签（P48）

    是 XML 格式的 Android 程序声明文件，包含了 Android 系统运行 Android 程序前所必须掌握的重要信息。这些信息包括应用程序名称，图标，包名称，模块组成，授权和 SDK 最低版本。

    根元素 ：

    包含 xmlns:android、package、xmlns:versionCode 和 android:versionName 四个属性

    1. xmlns:android —— 定义 Android 命名空间，值为 “http://schemas.android.com/apk/res/android"
    2. package —— 定义应用程序的包名称
    3. android:versionCode 定义应用程序的版本号，是一个整数值，值越大版本越新，仅在程序内部使用，并不提供给应用程序使用者
    4. android:versionName 定义应用徐的版本名称，是一个字符串，仅限用户提供一个版本标识

    manifuest 元素仅能包含一个 application 元素

  1. 什么是 R.java 文件，作用是什么（P49）

    R.java 文件是 ADT 自动生成的文件，包含对drawable、layout 和 values 目录内的资源的引用指针，Android 程序能够直接通过 R 类型引用目录中的资源。R.java 文件不能手工修改，所有代码必须由 ADT 自动生成。

  2. 4.1 程序生命周期，重点：安卓生命周期中包含哪五种生命进程，分别在什么时候执行或者产生，5种进程优先级是什么

    生命周期

    进程优先级

前台进程：是 Android 系统中最重要的进程，是与用户正在交互的进程：

  1. 进程中的 Activity 正在与用户进行交互
  2. 进程服务被 Activity 调用，而且这个 Activity 正在与用户进行交互
  3. 进程服务正在执行声明周期中的回调函数
  4. 进程的 BroadcastReceiver 正在执行 onReceive() 函数

可见进程：指部分程序界面能够被用户看见，却不在前台与用户交互，不响应界面事件的进程

  1. 新启动的 Android 程序将原有程序部分遮挡，原程序从前台进程变为可见进程
  2. 如果一个进程包含服务，且这个服务正在被用户可见的 Activity 调用，此进程通同样被视为可见进程

服务进程：包含已经启动的服务的进程就是服务进程

后台进程：不包含任何已经启动的服务，而且没有任何用户可见的 Activity，则这个进程就是后台进程

  1. 仅有 Activity 组件的进程，打那个用户启动了其他应用程序是这个进程的 Activity 完全被遮挡，这个进程便成为后台进程

空进程：是不包含任何活跃组件的进程

  1. 仅有 Activity 组件的进程，当用户关闭 Activity 后，这个进程就成为空进程

  1. 安卓系统中4大组件作用，哪四个（P67）

    1. 活动（Activity）

      作用：是 Android 程序的呈现层，显示可视化的用户界面，并接收与用户交互所产生的界面事件，用于提示用户程序已经正常启动

    2. 服务（Service）

      用于没有用户界面，但需要长时间在后台运行的应用

    3. 广播接收器（Broadcast Receiver）

      用来接收并响应广播消息的组件

    4. 内容提供器（Content provider）

      是 Android 系统提供的一种标准的共享数据的机制，应用程序可以通过 Content provider 访问其他应用程序的私有数据

  2. 4.3 节

    1. Activity四种状态以及定义

      1. 活动状态：当 Activity 在用户界面中处于最上层，完全能被用户看到，能够与用户进行交互
      2. 暂停状态：当 Activity 在界面上被部分遮挡，该 Activity 不再处于用户界面的最上层，且不能够与用户进行交互
      3. 停止状态：当 Activity 在界面上完全不能被用户看到，也就是说这个 Activity 被其他 Activity 全部遮挡
      4. 非活状态：不在以上三种状态中的就是非活动状态

    2. 生命周期中的回调函数（表4-1）有哪些，分别在什么时候被调用

      | 函数 | 是否可终止 | 说明 |
      | ———– | ———- | ———————————————————— |
      | onCreate() | 否 | Activity 启动后第一个被调用的函数，常用来进行 Activity 的初始化 |
      | onStart() | 否 | 当 Activity 显示在屏幕上时，该函数被调用 |
      | onRestart() | 否 | 当 Activity 从停止状态进入活动状态前，调用该函数 |
      | onResume() | 否 | 当 Activity 可以接收用户输入时，该函数被调用 |
      | onPause() | 否 | 当 Activity 进入暂停状态是，该函数被调用 |
      | onStop() | 是 | 当 Activity 不对用户可见后，该函数被调用，Activity 进入停止状态 |
      | onDestory() | 是 | 在 Activity 被终止前，即进入非活动状态前，该函数被调用：finish() 和被 Android系统终结 |

  1. P102-116 涉及到Android所有常用布局都需要掌握

    1. 线性布局特点，水平布局名称，垂直布局名称，控制方向的属性叫什么

      特点：所有子元素都在垂直或水平方向按照顺序在界面上排列

      控制方向属性：Orientation

      水平布局：horizontal

      垂直布局：vertical

    2. 框架布局的概念和标签

      概念：用来存放一个元素的空白空间，且子元素的位置是不能够指定的，只能放置在左上角，如果有多个元素，后放置的子元素将遮挡先放置的子元素

      标签：FrameLayout

    3. 表格布局的概念和标签

      概念：将屏幕划分为表格，通过指定行和列可以将界面元素添加到表格中，网格的边界对用户是不可见的，网格布局支持嵌套

      标签：TableLayout

    4. 相对布局的标签，属性

      概念：通过指定界面元素与其他元素的对应位置关系，确定界面中所有元素的布局位置

      标签：RelativeLayout

    5. 绝对布局的标签，特征和属性

      概念：通过指定界面元素的坐标位置，来确定用户界面的整体布局

      标签：AbsoluteLayout

    6. 网格布局和表格布局的区别

      网格布局（GridLayout）比表格布局在界面设计上更加灵活，在网格布局中界面元素可以占用多个网格，而表格布局无法实现，表格布局只能将界面元素指定在一个表格行中，不能跨越多个表格行

    快捷菜单类似于计算机程序中的邮件菜单，当用户点击界面上某个元素超过 2 秒后，将启动注册到该界面元素的快捷菜单，快捷菜单采用浮动的显示方式，启动方式与其他菜单不同

    需要用 registerForContextMenu() 函数将快捷菜单与指定的组件绑定或者说注册

  1. 界面事件的返回值对消息传递的影响机制（5.6.1节）

    控件设置监听器 -> 事件传递给监听器

    控件没有设置监听器 -> 事件传递给界面控件的其他事件处理函数

    界面控件设置了事件监听器，事件也可以再次传递给其他事件，是否继续传递时间给其他处理函数，是由监听器处理函数的返回值决定的，true -> 不需要其他处理函数参与处理，false -> 需要其他处理函数参与处理
