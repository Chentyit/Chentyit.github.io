project.properties 文件：记录了 Android 工程相关设置

AndroidManifest.xml 文件：是 XML 格式的 Android 程序声明文件，包含了 Android 系统运行 Android 程序前所必须掌握的重要信息

什么是 R.java 文件，作用是什么（P49）

R.java 文件是 ADT 自动生成的文件，包含对drawable、layout 和 values 目录内的资源的引用指针，Android 程序能够直接通过 R 类型引用目录中的资源。R.java 文件不能手工修改，所有代码必须由 ADT 自动生成。

4.1 程序生命周期，重点：安卓生命周期中包含哪五种生命进程，分别在什么时候执行或者产生，5种进程优先级是什么

生命周期

进程优先级

前台进程：是 Android 系统中最重要的进程，是与用户正在交互的进程：

  1. 进程中的 Activity 正在与用户进行交互
  2. 进程服务被 Activity 调用，而且这个 Activity 正在与用户进行交互
  3. 进程服务正在执行声明周期中的回调函数
  4. 进程的 BroadcastReceiver 正在执行 onReceive() 函数

可见进程：指部分程序界面能够被用户看见，却不在前台与用户交互，不响应界面事件的进程

  1. 新启动的 Android 程序将原有程序部分遮挡，原程序从前台进程变为可见进程
  2. 如果一个进程包含服务，且这个服务正在被用户可见的 Activity 调用，此进程通同样被视为可见进程

服务进程：包含已经启动的服务的进程就是服务进程

后台进程：不包含任何已经启动的服务，而且没有任何用户可见的 Activity，则这个进程就是后台进程

  1. 仅有 Activity 组件的进程，打那个用户启动了其他应用程序是这个进程的 Activity 完全被遮挡，这个进程便成为后台进程

空进程：是不包含任何活跃组件的进程

  1. 仅有 Activity 组件的进程，当用户关闭 Activity 后，这个进程就成为空进程

安卓系统中4大组件作用，哪四个（P67）

  1. 活动（Activity）

    作用：是 Android 程序的呈现层，显示可视化的用户界面，并接收与用户交互所产生的界面事件，用于提示用户程序已经正常启动

  2. 服务（Service）

    用于没有用户界面，但需要长时间在后台运行的应用

  3. 广播接收器（Broadcast Receiver）

    用来接收并响应广播消息的组件

  4. 内容提供器（Content provider）

    是 Android 系统提供的一种标准的共享数据的机制，应用程序可以通过 Content provider 访问其他应用程序的私有数据

4.3 节

Activity四种状态以及定

活动状态：当 Activity 在用户界面中处于最上层，完全能被用户看到，能够与用户进行交互

暂停状态：当 Activity 在界面上被部分遮挡，该 Activity 不再处于用户界面的最上层，且不能够与用户进行交

停止状态：当 Activity 在界面上完全不能被用户看到，也就是说这个 Activity 被其他 Activity 全部遮挡

非活动状态：不在以上三种状态中的就是非活动状

（1）什么是Intent过滤器，机制是什么

Intent过滤器是一种根据Intent中的动作（Action）、类别（Categorie）和数据（Data）等内容，对适合接收该Intent的组件进行匹配和筛选的机制。Intent过滤器的作用是匹配数据类型、路径和协议，还包括可以用来确定多个匹配项顺序的优先级。每个Intent过滤器可以定义多个标签，程序开发人员可以使用自定义的类别，或使用Android系统提供的类别

（2）怎么在android配置文档中配置过滤器
  • 这个是用来设置默认用户界，动作是 android.intent.action.MAIN，类别是 android.intent.category.LAUNCHER
  • 以浏览的方式启动相应的 Activity，动作是 android.intent.action.VIEW，类别是 android.intent.category.DEFAULT，协议是 schemodemo，主机名称部分是 edu.hrbeu
（3） Intent解析的匹配规则是什么 P156

Intent解析的匹配规则是：

  1. Android系统把所有应用程序包中的Intent过滤器集合在一起，形成一个完整的Intent过滤器列表；
  2. 在Intent与Intent过滤器进行匹配时，Android系统会将列表中所有Intent过滤器的“动作”和“类别”与Intent进行匹配，没有指定“动作”的Intent过滤器可以匹配任何的Intent，但是没有指定“类别”的Intent过滤器只能匹配没有“类别”的Intent；
  3. 把Intent数据Uri的每个子部与Intent过滤器的标签中的属性进行匹配；
  4. 如果Intent过滤器的匹配结果多于一个，则可以根据在标签中定义的优先级标签来对Intent过滤器进行排序，优先级最高的Intent过滤器将被选择。以上每步中不匹配的都将被过滤掉。

7.1 全都要，用启动方式使用Service，启动和终止分别使用什么函数，有什么特点

  • 启动方式

    启动函数：Context.startService()

    终止函数：Context.stopService()

    特点：在启动方式中，启动Service的组件不能够获取Service的对象实例，因此无法调用Service中的任何函数，也不能够获取Service中的任何状态和数据信息

  • 绑定方式

    通过Context.bindService()建立服务链接，通过Context.unbindService()停止服务链接

    特点：绑定Service的组件可以调用Service中实现的函数，或直接获取Service中状态和数据信息。如果在绑定过程中Service没有启动，Context.bindService()会自动启动Service，而且同一个Service可以绑定多个服务链接，这样可以同时为多个不同的组件提供服务

Service两种生命周期的含义，以及起始和终止函数

完全生命周期从 onCreate() 开始到 onDestory() 结束，在 onCreate() 中完成Service的初始化工作，在 onDestory() 中释放所有占用的资源

活动生命周期从 onStart() 开始，但没有与之对应的“停止”函数，因此可以粗略地认为活动生命周期是以 onDestory() 标志结束

P206 8.2.3 资源文件当中RAW文件夹，XML文件夹区别

在应用程序编译和打包时，/res/raw 目录下的所有文件都会保留原有格式不变。而 /res/xml 目录下一般用来保存格式化数据的 XML 文件，则会在编译和打包时将 XML 文件转换为二进制格式，用以降低存储器空间占用率和提高访问效率，在应用程序运行时会以特殊的方式访问

8.4节 数据共享 什么是ConnectProvider特点，作用，创建步骤需要重载哪六个方法名称

ContentProvider（数据提供者）是应用程序之间共享数据的一种接口机制

作用：进程间进行数据交互 & 共享，即跨进程通信

六个方法名称：

  1. delete()
  2. insert()
  3. query()
  4. update()
  5. getType()
  6. onCreate()

前四个用来对shujj进行删除，添加，查询，修改

第 5 个用来返回指定 URI 的 MIME 数据类型

第 6 个一般用来初始化底层数据集合简历数据连接等工作
